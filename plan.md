1. 개요 (Project Overview)목적: 네이버/구글의 트렌드 데이터와 쇼핑 데이터를 결합하여 '현재 인기 상품'과 '미래 유망 상품'을 발굴하는 리서치 프로그램 구축.핵심 기술: API Data Extraction, Vector Embedding, Cosine Similarity, Momentum Scoring.2. 1단계: 데이터 추출 모듈 구축 (Current Focus)AI 에이전트와 가장 먼저 구현할 데이터 수집 파이프라인입니다.2.1 네이버 데이터랩 (Trend API)목표: 특정 카테고리 내 키워드들의 검색 트렌드 수집.데이터: 날짜별 검색량, 연령별/기기별 선호도.로직: 최근 30일간 검색량 가속도($Momentum$) 계산용 데이터 확보.2.2 네이버 쇼핑 (Search/Shopping API)목표: 키워드와 연관된 실제 상품 정보 수집.데이터: 상품명, 최저가, 카테고리(1~4단계), 리뷰 수, 판매처 정보.로직: 벡터 임베딩을 위한 '상품명 + 카테고리' 텍스트 뭉치 확보.2.3 데이터 저장 및 관리형식: pandas DataFrame 또는 SQLite (초기 단계는 .csv나 .json 권장).3. 구현 단계 (Implementation Roadmap)Phase 1: API 통신 기반 마련 (Initial Setup)[ ] API Key 설정 (.env 파일 관리) 및 인증 클래스 구현.[ ] 네이버 검색 API 호출 함수 제작 (Keyword $\rightarrow$ Product List).[ ] 네이버 데이터랩 API 호출 함수 제작 (Keyword $\rightarrow$ Time-series Data).Phase 2: 데이터 정제 및 스코어링 (Data Processing)[ ] 검색량 변화율 기반 '트렌드 스코어' 알고리즘 구현.[ ] 현재형(Volume 중심) vs 미래형(Velocity 중심) 필터링 로직 개발.Phase 3: 벡터화 및 매칭 (Vector Engine - To be updated)[ ] 텍스트 전처리 (한국어 형태소 분석).[ ] Embedding API (OpenAI 등) 연동.[ ] 벡터 유사도 기반 상품 추천 로직.4. 기술 스택 (Tech Stack)Language: Python 3.10+Library: requests, pandas, python-dotenv, numpyIDE Context: AI Agent (Cursor/Windsurf)를 사용하여 모듈화된 코드 생성 지향.5. 실행 규칙 (Agent Instructions)모든 코드는 객체지향(OOP) 방식으로 작성하여 확장성을 고려할 것.API 호출 시 에러 핸들링(Rate Limit 등)을 반드시 포함할 것.데이터는 data/raw/ 및 data/processed/ 폴더로 구분하여 저장할 것.
